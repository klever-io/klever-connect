# Contract Examples

Interact with deployed Klever smart contracts and deploy new ones.

## Examples

### `call-contract.ts` — Query and invoke contract functions

Read state with `contract.call()` (free) and change state with `contract.invoke()` (transaction).

```bash
PRIVATE_KEY=your_hex_key CONTRACT_ADDRESS=klv1... npx tsx call-contract.ts
```

### `deploy-contract.ts` — Deploy a compiled WASM contract

Load a `.wasm` + `.abi.json` build artifact and deploy it to the network.

```bash
PRIVATE_KEY=your_hex_key WASM_PATH=./output/my.wasm ABI_PATH=./output/my.abi.json npx tsx deploy-contract.ts
```

## Key Concepts

### Contract instance

```typescript
import { Contract } from '@klever/connect-contracts'

// With signer (enables mutable calls)
const contract = new Contract(address, abi, wallet)

// With provider only (read-only)
const contract = new Contract(address, abi, provider)
```

### Readonly vs mutable calls

| Method                         | Sends tx? | Use case                 |
| ------------------------------ | --------- | ------------------------ |
| `contract.call(fn, ...args)`   | No        | View functions, getters  |
| `contract.invoke(fn, ...args)` | Yes       | State-changing functions |

### ABI format

Klever contracts use a JSON ABI file generated by `ksc build`:

```json
{
  "name": "MyContract",
  "endpoints": [
    {
      "name": "getCount",
      "mutability": "readonly",
      "inputs": [],
      "outputs": [{ "type": "u64" }]
    },
    {
      "name": "increment",
      "mutability": "mutable",
      "inputs": [{ "name": "amount", "type": "u64" }],
      "outputs": []
    }
  ],
  "types": {}
}
```

### Building contracts

Install `ksc` (Klever Smart Contract compiler):

```bash
# See https://docs.klever.org/smart-contracts/getting-started
ksc build
```

This produces `output/<name>.wasm` and `output/<name>.abi.json`.
